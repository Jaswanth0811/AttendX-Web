Vercel Root Directory Configuration Issue Detected

If you are seeing this page, it means Vercel is deploying the root folder of your repository instead of the frontend subfolder. This is why you got a 404 error before (since there was no homepage in the root).

To fix this and load your Next.js application, follow these simple steps:

  1. Go to your Vercel Dashboard.
  2. Select the attend-x-web-frontend project.
  3. Click on the Settings tab at the top of the project page.
  4. Under the General tab, look for the Root Directory section.
  5. Set the value to frontend.
  6. Click Save.
  7. Go to the Deployments tab, click the three dots next to your latest deployment, and select Redeploy (or push a new commit).

Once you change that, your Next.js application will compile and serve from the root URL correctly!
